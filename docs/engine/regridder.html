WeatherEngine - Game-ready weather simulations

Following the storm: the adaptive grid

A storm is tiny compared to the sky around it. Simulating your whole domain at storm-resolving 200 m cells would waste almost all of that work on empty, calm air. The regridder is the engine’s answer: it keeps fine cells only where the storm is and lets everywhere else stay coarse - and it moves that fine patch as the storm grows and drifts. This is what “adaptive resolution” means, and it is the feature that makes large, detailed weather affordable in a game.

You only need this if your GridConfig has levels > 1. With levels = 1 the grid is uniform and there is nothing to adapt.

The idea: refine where it matters, coarsen where it does not

Recall from Building a world that with levels > 1 the grid can sit at a coarse cell size and refine toward dx_finest in patches. The regridder decides, every so often, which patches deserve refinement:

  • Where the air is doing something interesting (a strong updraft, warm buoyant air, dense cloud, rotation), it splits coarse bricks into finer ones.
  • Where things have gone quiet, it merges fine bricks back into coarse ones to reclaim the cost.

The result is a grid that hugs the storm - fine over the updraft, coarse over the calm - and follows it around the domain. You run it periodically during advance, and it reports what it did.

Running it: advance’s second argument

The regridder runs inside advance, on a cadence you set:

// Advance 5 steps, and run the regridder every 5 steps (so once here).
client->advance(/*steps=*/5, /*regrid_every=*/5);
  • regrid_every = 0 disables regridding for that call (pure dynamics).
  • regrid_every = N runs the regridder every N base steps during the advance. Each run emits a Regridded event; the whole advance ends with one Advanced event.

Regridding is not free, so you do not do it every single step. Every few steps is plenty - the storm does not teleport between ticks.

Before it can run, you must install a configuration with set_regridder.

Configuring it: new_regridder and tiers

A regridder is a ladder of tiers. Each tier grants one more level of refinement, and says what signal justifies refining to that level. You build it, add tiers finest-last, and hand it over:

Box<Regridder> regridder = new_regridder(/*hold_rounds=*/3, /*min_age_rounds=*/3);

TierTrigger storm_birth{};        // tier 0 -> grants level 1
storm_birth.theta_above      = 0.8f;      // refine where |theta'| >= 0.8 K
storm_birth.w_above          = 1.5f;      // ...or |w| >= 1.5 m/s
storm_birth.condensate_above = 1.0e-4f;   // ...or cloud+rain >= 1e-4 kg/kg
storm_birth.zeta_above       = 1.0e30f;   // vorticity: effectively disabled
storm_birth.pad_bricks       = 1;         // refine 1 brick beyond the signal
storm_birth.max_bricks       = 320;       // cap this tier's fine bricks
regridder->add_tier(storm_birth);

TierTrigger storm_core{};         // tier 1 -> grants level 2 (finest)
storm_core.theta_above      = 1.0e30f;    // disabled
storm_core.w_above          = 5.0f;       // only strong updrafts
storm_core.condensate_above = 3.0e-3f;    // ...or heavy condensate
storm_core.zeta_above       = 5.0e-3f;    // ...or real rotation
storm_core.pad_bricks       = 1;
storm_core.max_bricks       = 256;
regridder->add_tier(storm_core);

client->set_regridder(std::move(regridder));
pump_until(*client, SimEventKind::Status);   // install ack

The tier ladder

  • The first tier you add grants level 1, the second grants level 2, and so on. So add them from coarsest trigger to finest. Level 0 (the base coarse grid) is always present and never capped.
  • A tier fires wherever any of its enabled signals crosses its threshold. A cell that meets a finer tier’s trigger gets refined all the way to that tier’s level.
  • The pattern above is the common one: a loose first tier catches the whole storm and its surroundings at medium resolution; a strict second tier drops the very finest cells only on the violent core (strong updraft, heavy condensate, real rotation).

TierTrigger signals

Each is a threshold; set a signal to a huge value (like 1e30f) to disable it. All map to concepts from Part I:

  • theta_above (K) - fire where the potential-temperature perturbation |theta'| is at least this. Marks warm buoyant air. See Heat.
  • w_above (m/s) - fire where the updraft/downdraft |w| is at least this. Marks active convection. This is the workhorse trigger.
  • condensate_above (kg/kg) - fire where cloud+rain water is at least this. Marks the visible storm body. See Water.
  • zeta_above (1/s) - fire where vorticity (local spin) is at least this. Vorticity is how fast the air is rotating; a threshold here refines the rotating core of a supercell - the mesocyclone from shear. Values around 5e-3 1/s mark real rotation.

pad_bricks and max_bricks

  • pad_bricks - refine this many bricks beyond where the signal actually is, as a margin. Because the storm moves between regrids, a pad of 1-2 bricks keeps it from outrunning its fine grid before the next run. Too small and the storm’s leading edge crosses onto coarse cells; too big and you waste fine cells on empty air.
  • max_bricks - a hard cap on how many fine bricks this tier may use (0 = uncapped). This is your cost ceiling: it bounds the worst-case work per step even if the storm grows huge. If the demand exceeds the cap, the regridder demotes the least-important regions to stay under budget.

Hysteresis: hold_rounds and min_age_rounds

If the grid refined and coarsened the instant a signal wobbled across a threshold, it would “flicker” - constantly splitting and merging the same bricks, which is wasteful and can jitter the solution. new_regridder takes two hysteresis counts (in regrid rounds) to damp this:

  • hold_rounds - after a brick’s signal drops below threshold, it must stay quiet this many rounds before it is allowed to coarsen. Prevents a briefly-calm patch from de-refining and then immediately needing to refine again.
  • min_age_rounds - a freshly refined brick must live at least this many rounds before it may coarsen. Prevents split-then-immediately-merge churn.

3 and 3 (as in the example) are sensible starting values. Raise them if you see the grid thrashing; lower them if the grid is too slow to release resolution behind a fast-moving storm.

Reading Regridded: did it keep up?

Each regrid emits a Regridded event with a RegridSummary. The field to watch is min_margin:

case SimEventKind::Regridded: {
    RegridSummary r = event->regrid_summary();
    if (r.min_margin == 0) {
        std::printf("warning: storm reached the edge of its fine grid!\n");
    }
    break;
}
  • min_margin - the distance (in finest bricks) from the nearest signal to the boundary of its fine region. It is your early-warning gauge:
    • -1 - nothing triggered at all (no storm, or thresholds too high).
    • large positive - comfortable; the fine grid surrounds the storm with room to spare.
    • small positive - the storm is nearing the edge of its fine patch.
    • 0 - failure: the storm has reached (and is spilling past) its fine grid. Its leading edge is now on coarse cells and detail is being lost. Fix by increasing pad_bricks, regridding more often (smaller regrid_every), or - if it keeps happening because the storm is fast - parking the storm.

The summary also reports splits (leaves refined), merges (leaves coarsened), and demoted_regions (patches the max_bricks budget forced back to coarse). Rising demoted_regions means you are hitting your cost cap

  • raise max_bricks if you can afford it, or accept the coarser edges.

When you do not need a regridder

  • Uniform grid (levels = 1): there is nothing to adapt; do not set one.
  • Small fixed domains where you can afford fine cells everywhere: simpler to run uniform.
  • Prototyping: get the storm right first with regrid_every = 0, then add adaptation for performance.

Takeaways

  • The regridder keeps fine cells on the storm and coarse cells elsewhere, and follows the storm - the core of affordable large-domain weather. Only relevant when levels > 1.
  • Configure a ladder of tiers with new_regridder + add_tier (finest-last). Each tier grants one level and fires on any enabled signal: theta_above, w_above, condensate_above, zeta_above.
  • pad_bricks gives the moving storm margin; max_bricks caps cost.
  • Hysteresis (hold_rounds, min_age_rounds) stops the grid from flickering; 3/3 is a good start.
  • Run it via advance(steps, regrid_every); watch min_margin in the Regridded event - 0 means the storm outran its fine grid.
