WeatherEngine - Game-ready weather simulations

Recipes

Ready-made setups you can adapt. Each says what weather it makes, which meteorology chapter it leans on, and the code that matters. They assume the pump_until helper and using namespace lines from the examples (see quickstart.cpp in the examples). Full programs live in the examples/ folder - these recipes are the decisions, not the boilerplate.

The boilerplate, once

Every recipe drops into this shape:

Box<SimClient> client = new_sim_client(0);      // or a core budget, e.g. 8

// 1) build a sounding (varies per recipe)
Box<Sounding> sounding = /* ... */;

// 2) build the world
GridConfig config{};   /* ... set fields ... */
AxisConfig axis{};     /* ... set fields ... */
SimOptions options = default_sim_options();
options.sound_speed = 120.0f;
options.acoustic_substeps = 7;
client->create_sim(config, axis, std::move(sounding), options);
if (pump_until(*client, SimEventKind::SimReady)->kind() == SimEventKind::Error) return;

// 3) install a scenario (varies per recipe)
// 4) optionally set a regridder (varies)
// 5) each frame: advance + grab + drain poll()

Below, only the parts that change are shown.

1. Smallest storm (start here)

Makes: one thunderstorm in a small box. Leans on: convection, scenarios.

A uniform 12.8 km box, a storm-ready sounding, one warm moist seed, no regridder.

Box<Sounding> sounding = sounding_convective();

GridConfig config{};
config.dx_finest = 200.0;  config.levels = 1;        // uniform 200 m
config.brick_size = 8;     config.halo_width = 4;
config.base_bricks = VoxelDims{8, 8, 0};
config.origin = Vec2{-6400.0, -6400.0};              // 12.8 km, centered

AxisConfig axis{};
axis.k_levels = 24;  axis.dz_surface = 200.0;
axis.dz_top = 600.0; axis.max_growth = 0.10;

// ... create_sim ...

Box<Scenario> scenario = new_scenario();
Vec3 center{0.0, 0.0, 1000.0};
Vec3 radii {2000.0, 2000.0, 1000.0};
scenario->add_seed_ellipsoid(Field::Theta, center, radii, 3.0f);   // +3 K
scenario->add_seed_ellipsoid(Field::Vapor, center, radii, 0.003f); // +3 g/kg
client->install_scenario(std::move(scenario));
pump_until(*client, SimEventKind::Status);

for (int i = 0; i < 60; ++i) {
    client->advance(10, 0);                            // no regrid
    pump_until(*client, SimEventKind::Advanced);
}

Tune: weak storm -> moister sounding or bigger vapor seed. Nothing fires -> bigger theta seed (the cap is holding it down).

2. A parked supercell

Makes: a long-lived, rotating storm that stays centered while the world flows past. Leans on: shear.

The trick is subtracting the Bunkers storm motion from the wind so the storm does not walk off the grid.

Box<Sounding> sounding = sounding_convective();   // already has good shear

// Park it: shift into the storm's own moving frame.
Vec2 motion = sounding->bunkers_right_mover();

// Re-author the wind profile minus that motion. (Use your own levels;
// these mirror the built-in convective profile.)
WindPoint wind[] = {
    {    0.0f, -8.0f, -6.0f},
    { 2000.0f, -2.0f, -1.0f},
    { 6000.0f,  8.0f,  1.0f},
    {12000.0f, 12.0f,  2.0f},
    {20000.0f, 12.0f,  2.0f},
};
for (WindPoint& w : wind) {
    w.east  -= static_cast<float>(motion.x);
    w.north -= static_cast<float>(motion.y);
}
sounding->set_wind(Slice<const WindPoint>{wind, 5});

// Confirm the environment still looks supercellular AFTER parking
// (shear and helicity are frame-independent, so they should be unchanged):
SoundingReportData r = sounding->analyze();
std::printf("CAPE %.0f  SRH0-1 %.0f  SCP %.1f\n",
            r.cape, r.srh_0_1km, r.supercell_composite);

Use a bigger domain (levels = 3, base_bricks {16,16}) and a regridder (recipe 4) so the fine grid tracks the storm’s internal structure even while it sits still overall.

3. Adaptive domain that follows a moving storm

Makes: a storm free to drift across a large map, with fine cells chasing it. Leans on: the adaptive grid.

Do not park the storm; instead give it a big multi-level domain and a regridder.

GridConfig config{};
config.dx_finest = 200.0;  config.levels = 3;        // 800 m coarse -> 200 m fine
config.brick_size = 8;     config.halo_width = 4;
config.base_bricks = VoxelDims{16, 16, 0};
config.origin = Vec2{-51200.0, -51200.0};            // 102.4 km, centered

// ... create_sim, install a seed near one side ...

Box<Regridder> regridder = new_regridder(3, 3);       // hysteresis
TierTrigger birth{};                                   // level 1: the whole storm
birth.theta_above = 0.8f;  birth.w_above = 1.5f;
birth.condensate_above = 1.0e-4f; birth.zeta_above = 1.0e30f;
birth.pad_bricks = 1; birth.max_bricks = 320;
regridder->add_tier(birth);
TierTrigger core{};                                    // level 2: the violent core
core.theta_above = 1.0e30f; core.w_above = 5.0f;
core.condensate_above = 3.0e-3f; core.zeta_above = 5.0e-3f;
core.pad_bricks = 1; core.max_bricks = 256;
regridder->add_tier(core);
client->set_regridder(std::move(regridder));
pump_until(*client, SimEventKind::Status);

// Each frame: regrid every 5 steps, and WATCH THE MARGIN.
for (int frame = 0; frame < 300; ++frame) {
    client->advance(5, 5);
    for (;;) {
        Box<SimEvent> e = client->poll();
        if (e->kind() == SimEventKind::None) break;
        if (e->kind() == SimEventKind::Regridded &&
            e->regrid_summary().min_margin == 0) {
            std::printf("storm reached its fine-grid edge - raise pad_bricks\n");
        }
    }
}

Tune: min_margin hitting 0 -> raise pad_bricks or regrid more often. Cost too high -> lower max_bricks (accepts coarser edges).

4. A front: cold air undercutting warm air

Makes: storms firing along a boundary, not from a single bubble. Leans on: air masses.

Paint a cold, stable air mass into part of a warm, unstable domain. Where the cold air undercuts the warm moist air, it lifts it and fires convection along the seam.

Box<Sounding> warm = sounding_convective();   // domain base state
// ... create_sim with `warm` ...

Box<Sounding> cold = sounding_quiet();        // a cool, stable brush
Box<Scenario> scenario = new_scenario();

// Stamp cold air over the western half, as an initial condition (nudge 0),
// so it is free to advance eastward and undercut the warm air.
scenario->add_air_mass_box(
    *cold,
    Vec3{-51200.0, -51200.0,     0.0},
    Vec3{     0.0,  51200.0,  6000.0},
    /*softness=*/3000.0,
    /*with_winds=*/true,
    /*nudge_rate=*/0.0f,          // release it; let the front move
    envelope_always());

// A gentle push to get storms going right at the seam.
scenario->add_seed_ellipsoid(Field::Theta, Vec3{500.0, 0.0, 800.0},
                             Vec3{1500.0, 4000.0, 700.0}, 2.0f);
client->install_scenario(std::move(scenario));

Tune: set nudge_rate > 0 to hold the cold pool as a persistent source (continuous outflow) instead of a one-time front.

5. Reading the storm each frame (rendering + telemetry)

Makes: nothing new - shows the read-out loop. Leans on: getting data out.

ViewBox view{};
view.min = Vec2{-51200.0, -51200.0};
view.max = Vec2{ 51200.0,  51200.0};
view.resolution = VoxelDims{128, 128, 48};
ChannelPreset presets[] = {ChannelPreset::Cloud, ChannelPreset::Rain};

for (int frame = 0; frame < 600; ++frame) {
    client->advance(5, 5);
    client->grab_presets(view, Slice<const ChannelPreset>{presets, 2});

    // Cheap telemetry a few times a second, not every frame:
    if (frame % 30 == 0) {
        Field peak[] = {Field::W};
        client->query_max(Slice<const Field>{peak, 1});
        client->probe(Vec2{0.0, 0.0});
    }

    for (;;) {
        Box<SimEvent> e = client->poll();
        switch (e->kind()) {
        case SimEventKind::None: goto next_frame;
        case SimEventKind::Grabbed: {
            Box<ViewGrab> g = e->take_grab();
            // upload g->channel(0) (cloud), g->channel(1) (rain) as 3D textures
            client->recycle(std::move(g));       // always recycle!
            break;
        }
        case SimEventKind::Values:
            std::printf("peak |w| = %.1f m/s\n", e->values()[0]);
            break;
        case SimEventKind::Probed:
            std::printf("live CAPE %.0f\n", e->probe_report().cape);
            break;
        default: break;
        }
    }
    next_frame:;
}

6. Calm skies / a control run

Makes: deliberately nothing - a stable, quiet reference. Leans on: stability.

Use sounding_quiet() and either no scenario or only tiny perturbations. Weather refuses to happen in this environment - useful as a baseline, a clear-sky state, or to confirm your pipeline is wired up before you add a storm.

Box<Sounding> sounding = sounding_quiet();
// ... create_sim, no seeds (or a tiny one that fizzles), just step and grab ...

7. Save and restore (rewind / save game)

Leans on: snapshots.

// Save:
client->save_snapshot();
Box<SimEvent> snap = pump_until(*client, SimEventKind::Snapshot);
Slice<const uint8_t> bytes = snap->snapshot_bytes();
std::vector<uint8_t> saved(bytes.begin(), bytes.end());   // copy out now

// ... keep stepping ...

// Restore (rolls the sim back to the saved state):
client->load_snapshot(Slice<const uint8_t>{saved.data(), saved.size()});
pump_until(*client, SimEventKind::SimReady);
// Reinstall scenario / regridder here - a snapshot does not keep them.

Tuning cheat-sheet

SymptomFirst thing to try
Nothing firesBigger Theta seed; check analyze() CAPE > ~1000
Weak, mushy stormMoister sounding / bigger Vapor seed
Everything erupts at onceA bit more CIN (cap); smaller seed
Storm rains out fastMore shear in the wind profile
Storm walks off gridPark it (recipe 2) or regrid + bigger pad_bricks
min_margin hits 0Raise pad_bricks, lower regrid_every
create_sim Error (acoustic)Lower sound_speed, raise acoustic_substeps
Storm top looks clippedTaller domain (axis) so top clears the EL
Per-frame stutter/leakMake sure you recycle every grab
